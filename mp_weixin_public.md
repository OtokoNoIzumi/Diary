# 微信公众号发布手册（Diary 项目）

适用场景：快速发布“简单记录 + 配图”到公众号草稿箱。

## 0. 前置条件（只需确认一次）

- 已有凭据文件：`.baoyu-skills/.env`
  - `WECHAT_APP_ID=...`
  - `WECHAT_APP_SECRET=...`
- 已有默认配置：`.baoyu-skills/baoyu-post-to-wechat/EXTEND.md`

## 1. 最短发布流程（推荐）

1) 新建一篇 markdown（示例：`post_2026_05_06.md`）
2) 写 frontmatter（标题、作者、封面、原文链接、主题色）
3) 正文插入图片
4) 执行一条命令发草稿

## 2. 发布命令

在项目根目录执行：

```bash
npx -y bun ".cursor/skills/baoyu-post-to-wechat/scripts/wechat-api.ts" "post_2026_05_06.md"
```

## 3. 参数优先级（已在脚本内支持）

1. CLI 参数（最高）
2. markdown frontmatter
3. `EXTEND.md` 默认值
4. 脚本内默认值

常用字段：
- `title`
- `author`
- `description` / `summary` / `digest`
- `coverImage`
- `sourceUrl`（"阅读原文"链接）
- `theme` / `color`

## 4. 美食Skills 文章框架
这是日常高频更新的板块。核心原则：**重展示，轻说教。把正文当 Vlog 的旁白写。**

### 标题格式

```
菜名.skills
```

示例：`鳗鱼滑蛋饭.skills`、`番茄炒蛋.skills`

---

### Description / 开场句写法

这类“美食skills”正文是 vlog 感，description 也尽量同口吻：
**description 不是摘要，是读者看到的第一句话**。
写的时候假设自己在群里发消息，有一些钩子的效果，而不是在写报告。

✅ 推荐：
```
点了份豪华外卖，为了算准热量，我硬是把食材全拆开单拍了……
```

❌ 不推荐（报告腔）：
```
今天这份鳗鱼滑蛋饭我完整记录了：3657kJ、蛋白37.6g...
```

---

### 正文结构（固定顺序）

```
**① 情绪化开场 + 海报图**
- 一句话交代今天的吃饭背景/心情。
- 直接放生成的高颜值精美海报图。

**② 真信息（Vlog 视角） + 软件截图**
- **只写截图里“最有趣/最出乎意料/最反常规”的一个点**（比如：饭只吃了1/4、把所有食材挑出来单拍、AI给的奇葩/精准建议）。
- **坚决不要**在文字里复述截图上已经有的热量、蛋白具体克数，读者长了眼睛自己会看。
- 放入对应的操作或数据截图。

**③ 💡 今日份 Diet Skill（使用技巧）**
- 提取今天记录过程中的一个软件技巧。
- 用“抛出痛点 -> 给出方案”的格式。
- 语气：“不要自己瞎琢磨，直接用这招”。

**④ 引流（Call to Action）**
- 一句话平滑过渡到“阅读原文”，不啰嗦。
- 示例：`想给自己的三餐找个“电子嘴替”，点击下方「阅读原文」传张照片试试吧~`
```

---

### DRY 原则（避免重复）

- 截图里已经有的数据（能量/营养明细），正文不要再抄一遍。
- 正文只写“截图里没有、但读者需要知道”的内容。
- 一般每组截图配 1 句补充就够了。
---

### 开场句写法

不要解释"今天这篇是XXX风格"，不要预告自己要干什么。
直接描述那一刻发生了什么。

❌：今天这餐是鳗鱼滑蛋饭配虾仁白菜。先放成品图，今天这篇就是日常 vlog 式记录，不讲大道理。

✅：好久没吃鳗鱼滑蛋饭了，今天点了一份解解馋

---

### 截图使用规则

| 截图内容 | 对应位置 | 说明 |
|---|---|---|
| 生成的营养海报 | 开场图，放最前面 | 视觉主角 |
| 营养分析数据页 | 第二张 | 数据在图里，正文不重复 |
| AI 点评界面 | 第三张 | 最有意思的内容，不要跳过 |
| 历史记录 / Dashboard | 技巧后面 | 混脸熟用，不需要解说 |

> 截图的作用是让读者慢慢熟悉产品界面，不是图解正文。放就行，不用描述。

---

### 技巧怎么写

每篇只写一个真实用过的习惯，不用凑数。
写法：像在群里随手说一句，不像在教课。

❌：对于没吃过的高热量食物，比起吃完了再拍照分析，完全可以吃一半就拍，再根据情况看还可以吃多少。

✅：遇到鳗鱼饭这种热量高，且我没记录过的食物时，我通常不会等全吃完了才去记录。我现在习惯在吃了一半的时候，就拍照分析。
提前看看还剩多少热量和碳水空间。如果发现快到红线了，剩下的米饭我就果断不吃了。

---

### 写作风格要求

- **不解释自己在做什么**，直接做。不要出现"今天这篇"、"这次我记录了"这类元评论。
- **DRY**：截图里有的数字正文不再列，录入步骤不用写。
- **短句**，每段1-3句。段与段之间留白。
- **AI 点评截图一定要用**，是整篇最有实质内容的部分。

## 5. 可复制的 frontmatter 模板

```yaml
---
title: "菜名.skills"
description: "一句自言自语，有画面，不超过30字。"
author: "爱点外卖的泉酱"
coverImage: "new_mpwx_files/封面图文件名.jpg"
sourceUrl: "https://izumitech.com"
theme: "default"
color: "#d97757"
---
```

## 6. 发布后人工收尾（目前仍需手动）

在公众号后台草稿页确认：

1. 是否勾选"原创"
2. 是否开启"赞赏"
3. 原创勾选后作者展示是否符合预期
4. 排版和图片是否正常

> 说明：原创/赞赏开关通常属于平台侧流程，不属于当前草稿接口可稳定自动化字段。

---

## 7. 常见问题

- 报 `40164 invalid ip not in whitelist`
  → 去公众号开发配置添加当前出口 IP 到 API 白名单。

- 颜色看起来变化不明显
  → `theme/color` 影响 markdown 渲染层；若正文以图片为主，视觉主要由图片本身决定。

- 想要网页级复杂交互（图表悬停等）
  → 公众号不支持 JS 交互，需导出静态图后插入正文。
